# 🤝 Contributing to MealMate AI

First of all, thank you for your interest in contributing to **MealMate AI**! ❤️

We welcome contributions from developers, designers, AI enthusiasts, students, nutrition experts, and open-source contributors who want to improve AI-powered meal planning and make healthy eating more accessible.

Whether you're fixing a bug, improving the UI, optimizing performance, or suggesting new features, every contribution is appreciated.

---

# 🌟 Ways to Contribute

You can contribute in many ways, including:

- 🐞 Fixing bugs
- ✨ Adding new features
- 🎨 Improving UI/UX
- ⚡ Optimizing performance
- 🤖 Enhancing AI prompts
- 📱 Improving responsiveness
- ♿ Improving accessibility
- 🔒 Enhancing security
- 📄 Improving documentation
- 🧪 Writing tests
- 💡 Suggesting ideas and improvements

---

# 🚀 Getting Started

## 1. Fork the Repository

Click the **Fork** button in the top-right corner of this repository.

---

## 2. Clone Your Fork

```bash
git clone https://github.com/your-username/mealmate-ai.git
```

---

## 3. Navigate into the Project

```bash
cd mealmate-ai
```

---

## 4. Install Dependencies

```bash
npm install
```

---

## 5. Start the Development Server

```bash
npm run dev
```

---

# 🌿 Create a New Branch

Always create a new feature branch before making changes.

```bash
git checkout -b feature/your-feature-name
```

Example:

```bash
git checkout -b feature/improve-pdf-export
```

or

```bash
git checkout -b fix/responsive-layout
```

---

# 💻 Coding Standards

Please follow these guidelines while contributing.

### Code Quality

- Write clean and readable code.
- Follow the existing project architecture.
- Keep components small and reusable.
- Avoid duplicate code.
- Remove unused imports.
- Remove unused variables.
- Use meaningful variable and function names.
- Keep functions focused on a single responsibility.

---

### UI Guidelines

Maintain the existing design language.

Do not change the design system unless necessary.

Use

- Matcha Green
- Cherry Red

Maintain

- Responsive Layout
- Accessibility
- Consistent Spacing
- Smooth Animations

---

### Performance

Please ensure your code:

- Avoids unnecessary re-renders
- Uses reusable components
- Optimizes expensive operations
- Does not reduce application performance

---

### Accessibility

Every contribution should support:

- Keyboard navigation
- Semantic HTML
- Proper labels
- Focus indicators
- Color accessibility

---

# ✅ Testing

Before creating a Pull Request, please verify:

- Application builds successfully
- No console errors
- No runtime errors
- Responsive on mobile and desktop
- Buttons work correctly
- PDF export works
- AI generation works
- Forms validate correctly

---

# 📝 Commit Messages

Use meaningful commit messages.

Examples:

```text
feat: add grocery list export

fix: resolve PDF generation issue

docs: improve project documentation

refactor: optimize meal generation logic

style: improve responsive layout

perf: optimize rendering performance
```

---

# 📤 Pull Request Process

Before submitting a Pull Request:

- Sync with the latest main branch.
- Test your changes.
- Update documentation if required.
- Ensure no unnecessary files are included.

When opening a Pull Request, include:

- What changed
- Why it changed
- Screenshots (if UI changes)
- Testing performed

---

# 🐞 Reporting Bugs

When reporting bugs, please include:

- Operating System
- Browser
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)

---

# 💡 Feature Requests

Feature requests are welcome.

Please include:

- The problem
- Proposed solution
- Expected user benefit
- Optional mockups or examples

# 🔒 Security

Do not expose API keys.

Never commit:

- .env files
- Private credentials
- Secrets
- Tokens

If you discover a security issue, please report it responsibly instead of publicly disclosing it.
